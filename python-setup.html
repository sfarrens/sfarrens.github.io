Seting up Python on Mac OSX

Contents

  • Installation
  • Environment

This objective of this post is to help you set up Python 2.7 on Mac OSX. I am currently running 10.12 (Sierra), but most of these suggestions should be applicable to any OS.

I may write another post for help with Python 3 in the future.

Installation

A default version of Python 2.7 is actually pre-installed on OSX.

$ which python
/usr/bin/python

You will find, however, that very few people reccommend using the default installation. While the default version certainly works, it can be cumbersome to install and manage packages. So, if you plan to develop in Python, you will probably want to install it yourself.

There are plenty of ways to install Python yourself, here I will simply focus on 3 easy methods.

1. Anaconda: Anaconda is an open source Python and R distribution platform (and my personal preference). You can download either a graphical installer (.pkg) or command line installer directly from their website. Anaconda comes with a lot of useful pre-installed packages and offers excellent management resources. If, however, you want to keep things as simple (clean) as possible, you may prefer one of the following options.

2. Homebrew: Homebrew is an open source package management system for OSX. To install Homebrew simply run the follow command:

$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

On the Homebrew website you can find out more about the various packages available. To install Python using Homebrew run the following:

$ brew install python

This will install a Python binary to /usr/local/bin.

3. MacPorts: MacPorts is an alternative package management system for OSX. Details for installing MacPorts are available on their website. To install Python using MacPorts run the following:

$ sudo port install python

This will install a Python binary to /opt/local/bin.

Environment

Now that you have installed Python you can set up your Python evnvironment so that your new Python becomes the default version and it can find all of your packages.
